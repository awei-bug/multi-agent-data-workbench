# Root Cause Tracing

当错误出现在深层调用栈时，不要停留在报错点本身，继续往上追。

## 方法

1. 找到报错点或异常表现点
2. 问：坏值、坏状态、错误调用是从哪里来的？
3. 继续向上追调用方、数据来源、状态来源
4. 直到找到第一个真正把错误引入系统的位置

## 在本仓库中的典型路径

- 组件异常 → props / hook / store selector → store state → mock/API 数据来源
- 页面不切换 → route state → 路由渲染边界 → 残留构建产物 / 错误入口
- API 响应异常 → 前端请求层 → 后端路由 → service / DB / 外部模型调用

## 目标

修“错误被引入的位置”，而不是修“错误第一次爆出来的位置”。
