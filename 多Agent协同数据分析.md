1\. 项目需求与目标定义

1.1 业务需求

支持上传结构化数据（Excel/CSV/ 数据库直连），无需人工写 SQL 即可完成清洗、统计、建模。

多个专业 Agent 分工协作，自动完成数据读取→清洗处理→指标计算→可视化绘图→分析解读→报告撰写全链路。

可视化工作台提供拖拽式交互，支持人工介入修正 Agent 输出、调整分析维度、补充业务逻辑。

支持多轮对话协同，人工下发分析需求，Agent 之间自动传递上下文、任务分片、结果互审。

输出产物：清洗后数据集、指标看板、图表、数据分析结论、Word/PDF 完整分析报告。

权限隔离：数据、任务、报告分用户 / 部门隔离，日志全留存可追溯。

1.2 技术目标

多 Agent 任务调度：串行 + 并行混合协同，支持任务依赖、重试、中断恢复。

数据安全：数据脱敏、查询权限、文件访问鉴权。

可扩展性：新增行业 Agent（财务 / 运营 / 供应链）无需重构底层调度。

低代码：业务人员仅自然语言描述需求即可生成完整数据分析成果。

1.3 交付边界

包含 Web 前端工作台、Agent 调度服务、数据处理中间件、文件存储、报告导出模块；不含底层大模型基座（可对接 OpenAI / 通义千问 / DeepSeek / 本地 LLM）。

2\. 整体系统架构设计

2.1 四层分层架构

plaintext

前端交互层 → 业务调度服务层 → 多Agent智能体层 → 数据资源底层

前端交互层（工作台 Web）

数据上传面板、任务创建画布、Agent 实时日志、图表预览、报告编辑导出、用户权限管理。

技术栈：Vue3/React + Ant Design/XDataV + ECharts/Plotly 可视化组件。

业务调度服务层（核心中枢）

Agent 任务调度引擎、上下文管理、消息队列、权限校验、文件中转、结果缓存、API 网关。

技术栈：Python FastAPI / Java SpringBoot + Redis + RabbitMQ 消息队列。

多 Agent 智能体层

独立封装 8 类专用 Agent，由调度中心统一分发任务、传递数据、校验输出。

数据资源底层

数据源：本地文件、MySQL/PostgreSQL、ClickHouse、API 业务数据库。

存储：MinIO 文件存储、MySQL 任务元数据表、Redis 临时缓存、向量库（存储历史分析案例）。

2.2 数据流架构简图

用户输入自然语言需求 + 上传数据 → 调度引擎解析任务图谱 → 分发至对应 Agent 集群 → Agent 串行 / 并行处理 → 结果回传工作台 → 人工校验 / 二次编辑 → 导出分析报告

3\. 多 Agent 角色分工与能力定义

3.1 总控调度 Agent（总指挥）

核心职责：解析用户自然语言需求，拆解子任务，生成任务依赖 DAG 图，分配给下游 Agent，统一汇总所有输出产物，控制任务启停、重试、超时熔断。

能力：需求语义理解、任务分片、上下文持久化、Agent 通信协调、异常兜底。

3.2 数据接入 Agent

核心职责：读取 CSV/Excel/ 数据库表，解析字段、识别数据类型、检测空值 / 乱码 / 格式错误，生成原始数据快照。

输出：结构化数据表、数据概况统计（行数、字段类型、缺失率）。

3.3 数据清洗 Agent

核心职责：去重、填充缺失值、异常值剔除、格式统一（日期 / 金额 / 编码）、字段重命名、数据脱敏、数据分片。

输出：清洗后标准数据集、清洗操作日志、异常数据明细。

3.4 指标计算 SQL Agent

核心职责：根据业务需求自动生成统计 SQL / 聚合逻辑，完成维度拆解（时间、部门、渠道），计算同比 / 环比 / 均值 / 占比等核心指标。

输出：可执行 SQL 脚本、指标结果数据集、指标口径说明文档。

3.5 可视化绘图 Agent

核心职责：根据指标数据自动匹配图表类型（折线 / 柱状 / 饼图 / 散点 / 热力图），配置坐标轴、图例、配色，生成可预览图表组件。

输出：ECharts 配置 JSON、图表图片、可视化看板布局。

3.6 分析推理 Agent

核心职责：基于指标 + 图表挖掘业务规律，定位波动原因、风险点、增长机会，输出客观数据解读，区分数据事实与业务推论。

输出：分层数据分析结论、风险预警、优化建议。

3.7 报告撰写 Agent

核心职责：整合清洗日志、指标、图表、分析结论，按标准模板生成完整结构化分析报告，支持财务 / 运营 / 市场多模板切换。

输出：Markdown 草稿、可导出 Word/PDF 完整报告。

3.8 审核校验 Agent

核心职责：全链路结果校验，检查数据逻辑冲突、指标口径不一致、图表错误、结论脱离数据事实，标记可疑内容返回对应 Agent 重算。

输出：校验报告、问题清单、重算任务指令。

6\. 数据流转与 Agent 协同调度逻辑

6.1 标准串行协同流程（常规分析任务）

用户在工作台上传数据 + 输入自然语言分析需求，提交创建任务。

前端通过 WebSocket 推送任务至调度服务，生成唯一 task\_id、context\_id。

总控调度 Agent 接收需求，解析并生成 DAG 任务链：

数据接入Agent → 数据清洗Agent → 指标计算Agent → 可视化Agent → 分析推理Agent → 报告撰写Agent → 审核校验Agent

调度中心通过 MQ 依次下发任务，前序 Agent 执行完成后，将数据集、日志存入 Redis 上下文，触发下一个 Agent 执行。

每一步 Agent 输出实时推送前端工作台，用户可随时查看中间结果，支持人工修改数据 / 指标后重跑下游所有 Agent。

全部 Agent 执行完成后，审核 Agent 校验全链路产物，若存在问题标记对应节点，用户可一键重算指定 Agent。

校验通过后，工作台展示完整数据集、看板、分析报告，支持导出文件。

6.2 并行协同场景（多图表、多维度分析）

总控 Agent 拆解多维度指标需求，清洗完成后并行分发多组指标计算 Agent。

各组指标结果独立送入可视化 Agent 并行绘图。

并行产物全部完成后统一汇总至分析 Agent 整合结论。

6.3 人工介入协同逻辑

任意 Agent 执行完成后，用户可在工作台修改中间数据 / 参数。

人工修改内容存入上下文，调度引擎自动识别下游依赖节点，仅重跑受影响 Agent，无需从头执行全流程，节约算力。

支持手动新增自定义 Agent 节点，插入任务流程中补充业务逻辑。

6.4 任务中断与恢复机制

每个 Agent 执行完成自动快照保存上下文数据；

服务宕机 / Agent 超时中断后，重新打开任务可从最近完成的 Agent 节点继续执行。

8\. 测试方案与验证标准

8.1 功能测试标准

全链路流程通过率 ≥98%；

脏数据、缺失数据场景 Agent 可自动处理并输出异常说明；

人工修改中间数据后，下游 Agent 可正确重算；

审核 Agent 可识别明显数据逻辑错误并标记。

8.2 性能测试标准

10 万行 CSV 完整分析全流程耗时 ≤3 分钟；

单任务并发支持 50 个用户同时运行无阻塞；

WebSocket 日志推送延迟 <500ms。

8.3 安全测试标准

不同用户无法访问他人任务、数据、报告；

敏感字段自动脱敏，导出文件不泄露原始隐私数据；

禁止 SQL 注入、文件上传漏洞、越权访问接口。

9\. 部署方案

9.1 本地单机部署（测试环境）

Docker Compose 一键启动：前端、调度服务、8 类 Agent、Redis、RabbitMQ、MinIO、MySQL 全部容器化，适合开发调试。

9.2 云服务器分布式部署（生产环境）

网关 / 前端：Nginx 负载均衡；

调度服务集群：多实例水平扩容；

Agent 集群：按类型拆分部署，绘图 / 报告 Agent 消耗算力大，独立集群；

中间件：Redis 集群、RabbitMQ 集群、对象存储 OSS 替代 MinIO；

数据库：MySQL 主从备份，定时自动备份任务数据。

9.3 资源分配参考

调度服务：2 核 4G；

计算 / 绘图 / 报告 Agent：4 核 8G（LLM 调用算力消耗高）；

消息队列 / 缓存：4 核 8G；

存储：按需扩容对象存储容量。

10\. 运维、监控与迭代优化

10.1 监控指标

业务指标：任务创建量、任务成功率、各 Agent 失败次数、平均执行时长；

服务指标：CPU / 内存 / 磁盘使用率、接口 QPS、响应耗时；

存储指标：文件存储占用、数据库表容量、缓存命中率。

10.2 定时运维任务

每日凌晨自动备份 MySQL 任务数据；

30 天前过期临时文件自动清理；

每周更新案例向量库，优化相似任务匹配精度。

10.3 迭代优化方向

扩充行业专用 Agent（财务、供应链、人力）；

优化 Agent 提示词，降低分析结论偏差；

增加 AI 建模 Agent，支持预测分析、回归模型；

接入 BI 大屏，支持实时数据看板自动刷新。

11\. 风险点与解决方案

表格

风险点	解决方案

LLM 调用超时，Agent 卡死	设置接口超时熔断，自动重试 3 次，失败标记任务可手动重跑

超大文件处理内存溢出	文件分片读取、流式数据处理、临时落盘不加载全量数据至内存

Agent 输出数据逻辑错误	校验 Agent 强制前置校验，人工二次确认后才可导出正式报告

多任务并发算力不足	Agent 集群横向扩容，消息队列流量削峰

用户原始数据泄露	全程数据脱敏、接口鉴权、文件访问鉴权、操作日志全留存

任务上下文丢失	Redis 持久化 + MySQL 任务快照双备份
